1. Required Background Knowledge and Forward Expectations

Throughout this course, it is assumed that you have rudimentary programming skills that are taught in a Freshman or Sophomore level structured/logical coding class. A detailed understanding of digital logic design, both sequential and combinational, is required to complete the laboratory exercises.

During the course of this semester, you will conduct 12 laboratory experiments on the workstations in the ASIC Design Laboratory. Make sure to read through the lab before coming to class to have an idea of the goal and necessary steps.

2. Account Setup

2.1 Change Password

Log in to a workstation and enter the following command to change your password:

passwd

2.2 Initial Setup

Run the following command to copy the necessary setup files:

~ece337/setup337

Once done, open a new terminal window to apply the changes. Maintain an organized directory structure for your labs by creating separate directories for each lab assignment.

2.3 Git Setup

The setup script initializes an ece337 folder as a Git repository. If you have a remote repository, you can use it as your origin repository for version control. Ensure that these repositories remain private due to the nature of the course content.

2.4 Lab 1 Specific Setup

Create the directory for Lab 1 using the following command:

mkdir -p ~/ece337/Lab1

Run the dirset script to create the required subdirectories (Analyzed, Mapped, Reports, Schematic, Scripts, Source, Docs) for the lab.

2.5 Initial Git Commit

Add and commit the Lab 1 directory to Git using the following commands:

git add ~/ece337/Lab1
git commit -m "Initial setup for Lab 1"

3. Conventions

3.1 File Names

Design module source files must be named after the module (e.g., counter.sv for a module named counter). Testbench files must prepend tb_ to the module name (e.g., tb_counter.sv).

4. Compiling Verilog Code

Launch QuestaSim® using the following command:

vsim -i

Create a work directory and compile your Verilog source code:

vlib work
vlog source/comparator.sv

5. Simulating the Verilog Code

5.1 Simulation Setup

Start a new simulation session and select the comparator module from the work library. Add input and output signals to the waveform window.

5.2 Forcing Signals

Use the following commands to force inputs to the comparator:

force a 16#0000 0 ns, 16#ABCD 25 ns, 16#8808 50 ns -repeat 75 ns
force b 16#0000 0 ns, 16#9876 25 ns, 16#AAAA 50 ns -r 75 ns

6. Synthesizing the Comparator

Synthesize your design using the following command:

make synthesis

Check the log files for errors or warnings before proceeding.

8. Digital Design Refresher (Post-lab)

Complete the following tasks as part of the digital design refresher:

  • Generate a truth table for the sensor detector circuit.
  • Create K-Maps to determine the optimal logic circuit.
  • Create schematic diagrams for synchronizers, shift registers, and the 1101 detector.
